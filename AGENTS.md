# Project Instructions

This file provides context for AI assistants working on this project.

## Project Type: Tauri v2 Desktop App (React + Rust)

A PC 2FA (Two-Factor Authentication) authenticator app. Frontend is React 19 + TypeScript running on Vite, backend is Rust with Tauri v2.

### Stack
- **Frontend**: React 19, TypeScript 6, SCSS, wouter (routing)
- **Backend**: Rust, Tauri v2.11, Serde
- **Build**: Vite 8, tsc, Tauri CLI
- **Styling**: SCSS Modules (`*.module.scss`)
- **Formatter**: Prettier

### Commands

| Command | Description |
|---|---|
| `npm run dev` | Start Vite dev server only |
| `npm run build` | TypeScript check + Vite build |
| `npm run preview` | Preview Vite build |
| `npm run tauri` | Run Tauri CLI directly |
| `npm run start` | Start full Tauri dev mode (frontend + native window) |

### Project Structure

```
src/                # React frontend source
├── assets/         # SVG icons, images
├── components/     # Reusable UI components
│   └── Code/       # 2FA code display component
├── pages/          # Route pages
│   └── Setting/    # Settings page
├── App.tsx         # Root component with routing
├── App.css         # Global styles
└── main.tsx        # Entry point

src-tauri/          # Rust backend source
├── src/main.rs     # Tauri app entry, IPC commands
├── Cargo.toml      # Rust dependencies
├── tauri.conf.json # Tauri configuration
└── capabilities/   # Permission capabilities
```

### Routes (via wouter)

- `/` — Code display page (TOTP codes with countdown timer)
- `/setting` — Settings page (manage accounts)

### Tauri Configuration Notes

- Window: 360×600, non-resizable, centered
- Dev URL: `http://localhost:1420`
- Frontend is served from `dist/` in production build
- Build hooks: `beforeDevCommand` → `npm run dev`, `beforeBuildCommand` → `npm run build`
- CSP is disabled (`null`) for development flexibility

### Path Aliases

- `@/` → `src/` (configured in both Vite and TypeScript)

## Language

- **Preferred language**: 中文（简体）
- **代码注释**: 必须使用中文编写注释，解释代码逻辑、函数用途、参数说明等
- **文档**: 所有技术文档、README、API 说明、变更日志等，均使用中文撰写
- **提交信息**: Git commit message 使用中文描述变更内容
- **UI 文本**: 用户界面显示的文本使用中文（如无特殊要求）
- **变量/函数命名**: 代码标识符仍使用英文（camelCase / PascalCase），仅注释和文档用中文

## Conventions

- **Components**: Functional components with `FC` type, named exports
- **Styling**: SCSS Modules imported as `styles` (e.g., `import styles from './index.module.scss'`)
- **Path imports**: Use `@/` alias for cross-directory imports
- **TypeScript**: Strict mode enabled; `noUnusedLocals` and `noUnusedParameters` are errors
- **Naming**: camelCase for variables/functions, PascalCase for components
- **Assets**: SVG icons placed in `src/assets/`, imported directly

## Development Workflow

1. Start Tauri dev mode: `npm run start` (launches native window with hot reload)
2. Frontend-only development: `npm run dev` (browser-only at `localhost:1420`)
3. Build for distribution: `npm run build` followed by `npm run tauri build`

## Guidelines

- Follow existing SCSS Module patterns — do not use plain CSS or CSS-in-JS
- Use `wouter` for routing, not `react-router`
- New IPC commands must be registered in `tauri::generate_handler![]` in `main.rs`
- Keep Tauri permissions minimal in `capabilities/default.json`
- Write tests for new functionality
- Keep changes focused and atomic
- Document public APIs

## Important Notes

- The app's Tauri identifier is `com.tauri.dev` — update before production release
- Rust backend currently has an example `greet` command that can be removed or repurposed
- `src-tauri/gen/schemas/` is gitignored (auto-generated by Tauri)
